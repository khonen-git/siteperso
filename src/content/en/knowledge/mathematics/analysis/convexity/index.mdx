# Convexity

<MdxCard title="Definition">
A set <MathInline>{"C \\subseteq \\mathbb{R}^n"}</MathInline> is **convex** if, for all <MathInline>{"x, y \\in C"}</MathInline> and all <MathInline>{"\\lambda \\in [0,1]"}</MathInline>:

<MathBlock>{"\\lambda x + (1-\\lambda)y \\in C"}</MathBlock>

A function <MathInline>{"f : C \\to \\mathbb{R}"}</MathInline> is **convex** if its graph lies above every chord joining two points:

<MathBlock>{"f(\\lambda x + (1-\\lambda)y) \\leq \\lambda f(x) + (1-\\lambda)f(y)"}</MathBlock>

Convexity is **strict** if the inequality is strict whenever <MathInline>{"x \\neq y"}</MathInline> and <MathInline>{"\\lambda \\in (0,1)"}</MathInline>.
</MdxCard>

## Intuition

<MdxCard>
Geometrically, a convex set has no holes or indentations: the segment between any two points stays inside. A convex function curves upward: every chord between two graph points dominates the function.

This property is central in optimization: on a convex domain, any local minimum of a convex function is **global**. There are no local traps.
</MdxCard>

## Characterizations

<MdxCard title="Jensen's inequality">
If <MathInline>{"f"}</MathInline> is convex and <MathInline>{"X"}</MathInline> is a random variable with well-defined quantities:

<MathBlock>{"f(\\mathbb{E}[X]) \\leq \\mathbb{E}[f(X)]"}</MathBlock>

This is the standard tool linking averages and nonlinear transformations — useful in probability, finance, and risk analysis.
</MdxCard>

<MdxCard title="Gradient criterion (differentiable functions)">
<MathInline>{"f"}</MathInline> is convex if and only if, for all <MathInline>{"x, y"}</MathInline>:

<MathBlock>{"f(y) \\geq f(x) + \\nabla f(x)^\\top (y - x)"}</MathBlock>

The graph dominates its tangents: the linearization always underestimates the function.
</MdxCard>

<MdxCard title="Hessian criterion (twice differentiable)">
If <MathInline>{"f"}</MathInline> is <MathInline>{"C^2"}</MathInline>, it is convex on a convex open set if and only if:

<MathBlock>{"\\nabla^2 f(x) \\succeq 0 \\quad \\text{(positive semidefinite)}"}</MathBlock>

for every <MathInline>{"x"}</MathInline> in the domain. This is the most practical test for verifying convexity of a loss or objective function.
</MdxCard>

## Applications

<MdxCard>
Convexity is the foundation of **convex optimization**: problems where the objective and feasible set are convex. Classical algorithms (gradient, interior-point methods) then converge to the global optimum under moderate assumptions.

**Direct links on this site:**

- **[Linear regression](/knowledge/statistics/regression)** — least squares minimizes a strictly convex quadratic function (when <MathInline>{"X^\\top X"}</MathInline> is positive definite).
- **[Convex optimization](/knowledge/mathematics/optimization/convex-optimization)** — gradient descent, constraints, Lagrange multipliers; convexity ensures these methods target the right optimum.
- **Portfolio allocation** — Markowitz formulates a convex quadratic program under linear constraints.
- **Machine learning** — MSE, hinge loss, and cross-entropy (via the log) rely on convex structure in the parameters; this is what makes training linear models and SVMs reliable.

**Finance link (distinct):** **bond convexity** measures the curvature of a bond price with respect to yields (second derivative of price). It uses the language of convexity but is not the same topic as mathematical convexity — see [asset classes](/knowledge/quantitative-finance/markets-products/asset-classes).
</MdxCard>
