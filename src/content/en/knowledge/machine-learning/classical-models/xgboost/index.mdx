# XGBoost

<MdxCard title="Definition">
**XGBoost** (Extreme Gradient Boosting) is a **gradient boosting** method built on decision trees. Unlike the bagging used in [Random Forest](/knowledge/machine-learning/classical-models/random-forest), trees are trained **sequentially**: each new tree corrects part of the current model's errors.
</MdxCard>

## Intuition

<MdxCard>
The goal is not to average independent trees, but to build an **additive** ensemble:

- a first tree captures a coarse signal;
- following trees learn residual errors;
- the sum of trees produces a function more expressive than a single tree.

XGBoost adds **explicit regularization**, efficient handling of tabular data, and training optimizations that make it a practical reference on many structured supervised problems.
</MdxCard>

## How it works

<MdxCard>
  <ol className="list-decimal pl-6 space-y-3">
    <li><strong>Initialization</strong>: start from a constant prediction (mean for regression, log-odds for classification).</li>
    <li><strong>Gradient computation</strong>: at each iteration, measure the direction in which the loss should be corrected.</li>
    <li><strong>Weak tree</strong>: train a small tree on these gradients / pseudo-residuals.</li>
    <li><strong>Additive update</strong>: add this tree to the ensemble with a weight controlled by <code>learning_rate</code>.</li>
    <li><strong>Repeat</strong>: continue until <code>n_estimators</code> is reached or a stopping rule is triggered, ideally with <code>early_stopping</code>.</li>
  </ol>
</MdxCard>

<MdxCard>
In practice, XGBoost optimizes an objective made of:

- a **fit loss** (prediction error),
- **tree complexity** control (depth, leaves),
- **shrinkage** through <code>learning_rate</code>.

This explains why XGBoost can reach very strong performance, but also why it usually requires more tuning than a random forest.
</MdxCard>

## Key hyperparameters

## Strengths and limitations

<div className="grid gap-4 md:grid-cols-2">

<MdxCard>

### Strengths

- Very strong performance on structured tabular data
- Captures nonlinearities and complex feature interactions
- Rich regularization and fine control of the bias-variance trade-off
- Handles heterogeneous tabular features well
- Often outperforms [Random Forest](/knowledge/machine-learning/classical-models/random-forest) in raw predictive accuracy

</MdxCard>

<MdxCard>

### Limitations

- More tuning-sensitive and more prone to overfitting
- Less simple to explain and stabilize than a linear baseline or a forest
- Can behave poorly if validation is poorly designed
- Higher training cost, especially with hyperparameter search
- Like other tabular models, it does not solve <TermTip term="concept-drift">concept drift</TermTip> or <TermTip term="non-iid">non-iid</TermTip> issues by itself

</MdxCard>

</div>
