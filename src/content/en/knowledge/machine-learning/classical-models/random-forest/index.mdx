# Random Forest

<MdxCard title="Definition">
A **random forest** is an ensemble of **decision trees** trained on bootstrap samples of the data, with **random feature subsets** at each split. Final predictions aggregate trees by **majority vote** (classification) or **averaging** (regression).
</MdxCard>

## Intuition

<MdxCard>
A single tree captures nonlinear interactions but **overfits** easily (high variance). Random forest applies **bagging**: train many trees on different subsamples and average them to reduce variance without increasing bias as much.

Randomizing features at each split **decorrelates** trees — a necessary condition for aggregation to genuinely improve generalization over one deep tree.
</MdxCard>

## How it works

<MdxCard>
  <ol className="list-decimal pl-6 space-y-3">
    <li><strong>Bootstrap</strong>: for each tree <MathInline>b</MathInline>, draw <MathInline>n</MathInline> observations with replacement from <MathInline>n</MathInline> (about 63% unique observations per tree).</li>
    <li><strong>Unpruned tree</strong>: grow until depth or stopping criteria (<code>max_depth</code>, <code>min_samples_leaf</code>, etc.).</li>
    <li><strong>Random subspace</strong>: at each node, consider only a subset of <code>max_features</code> variables for the best split.</li>
    <li><strong>Aggregation</strong>: classification → majority class; regression → mean of <MathInline>B</MathInline> tree predictions.</li>
  </ol>
</MdxCard>

<MdxCard>
**Out-of-bag (OOB) score**: observations not in a tree's bootstrap sample act as an implicit validation set. With <code>oob_score=True</code>, you get a performance estimate without a dedicated validation set — useful for prototyping, but not a substitute for rigorous [cross-validation](/knowledge/machine-learning/general-concepts/cross-validation).
</MdxCard>

## Strengths and limitations

<div className="grid gap-4 md:grid-cols-2">

<MdxCard>

### Strengths

- Robust to outliers and feature scales (no mandatory normalization)
- Captures interactions and nonlinearities without explicit feature engineering
- Built-in feature importance; free OOB score
- Parallelizable; relatively few critical hyperparameters

</MdxCard>

<MdxCard>

### Limitations

- Less interpretable than a single tree
- Memory and latency grow with <code>n_estimators</code>
- Regression: poor extrapolation outside the training domain
- Can be outperformed by [gradient boosting](/knowledge/machine-learning/classical-models/xgboost) on structured tabular data
- Sensitive to **concept drift** and **non-iid** data without adaptation

</MdxCard>

</div>

## Applications

<div className="grid gap-4 md:grid-cols-2">

<MdxCard>

### Tabular classification

Credit scoring, fraud detection, multiclass classification on heterogeneous features — a historical sweet spot for random forests.

</MdxCard>

<MdxCard>

### Regression

Predicting continuous values (prices, volumes); watch for extrapolation outside the training support.

</MdxCard>

<MdxCard>

### Finance and markets

Baseline model for predicting a label (direction, regime) from technical features; often beaten by boosting but useful as a robust benchmark and for feature importance.

</MdxCard>

<MdxCard>

### Feature screening

Quickly identify relevant variables before a more complex model or [feature engineering](/knowledge/machine-learning/general-concepts) pipeline.

</MdxCard>

</div>

from sklearn.ensemble import RandomForestClassifier
from sklearn.model_selection import cross_val_score
from sklearn.inspection import permutation_importance

X_train, y_train = ...  # tabular features, labels

rf = RandomForestClassifier(
    n_estimators=200,
    max_depth=12,
    min_samples_leaf=5,
    max_features="sqrt",
    class_weight="balanced",
    oob_score=True,
    n_jobs=-1,
    random_state=42,
)
rf.fit(X_train, y_train)

print(f"OOB score: {rf.oob_score_:.3f}")
cv_scores = cross_val_score(rf, X_train, y_train, cv=5, scoring="f1_macro")
print(f"CV F1 macro: {cv_scores.mean():.3f} (+/- {cv_scores.std():.3f})")

# Impurity-based importance
for name, imp in sorted(
    zip(feature_names, rf.feature_importances_),
    key=lambda x: x[1], reverse=True,
)[:5]:
    print(f"{name}: {imp:.4f}")

# Permutation importance (on a hold-out)
