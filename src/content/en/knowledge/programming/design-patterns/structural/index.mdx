# Structural patterns

<MdxCard>
**Structural** patterns show how to assemble objects and classes into larger structures while keeping the design flexible and efficient.
</MdxCard>

## List

| Pattern | Key idea |
|---------|----------|
| [Adapter](/knowledge/programming/design-patterns/structural/adapter) | Wrap incompatible interfaces |
| [Bridge](/knowledge/programming/design-patterns/structural/bridge) | Decouple abstraction from implementation |
| [Composite](/knowledge/programming/design-patterns/structural/composite) | Part-whole tree |
| [Decorator](/knowledge/programming/design-patterns/structural/decorator) | Add behavior dynamically |
| [Facade](/knowledge/programming/design-patterns/structural/facade) | Simple API over a subsystem |
| [Flyweight](/knowledge/programming/design-patterns/structural/flyweight) | Share intrinsic state |
| [Proxy](/knowledge/programming/design-patterns/structural/proxy) | Controlled access substitute |
