# Visitor

<MdxCard>
Separate an algorithm from the classes it operates on.
</MdxCard>

## Intent

Add operations to a stable hierarchy without changing element classes.

## Python example

<CodeBlock language="python">
{`class Circle:
    def __init__(self, radius):
        self.radius = radius

    def accept(self, visitor):
        return visitor.visit_circle(self)

class Square:
    def __init__(self, side):
        self.side = side

    def accept(self, visitor):
        return visitor.visit_square(self)

class AreaVisitor:
    def visit_circle(self, circle):
        return 3.14 * circle.radius ** 2

    def visit_square(self, square):
        return square.side ** 2

shapes = [Circle(2), Square(3)]
visitor = AreaVisitor()
for s in shapes:
    print(round(s.accept(visitor), 2))`}
</CodeBlock>

<ConsoleOutput title="Console output">
12.56
9
</ConsoleOutput>

## When to use

New operations often, object structure changes rarely.
