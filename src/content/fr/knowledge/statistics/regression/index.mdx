# Régression linéaire

<MdxCard title="Définition">
La **régression linéaire** modélise la relation entre une variable cible <MathInline>{"y"}</MathInline> et un ensemble de variables explicatives <MathInline>{"x_1, \\ldots, x_p"}</MathInline> via une combinaison linéaire des coefficients.

Dans le cas multiple :

<MathBlock>{"y = X\\beta + \\varepsilon"}</MathBlock>

où <MathInline>{"X"}</MathInline> contient les features, <MathInline>{"\\beta"}</MathInline> les coefficients et <MathInline>{"\\varepsilon"}</MathInline> le terme d'erreur.
</MdxCard>

## Intuition

<MdxCard>
La régression linéaire cherche le **meilleur hyperplan** pour approximer une cible continue à partir des variables d'entrée. Le critère standard consiste à minimiser la somme des carrés des résidus.

Dans le cas simple :

<MathBlock>{"y_i = \\beta_0 + \\beta_1 x_i + \\varepsilon_i"}</MathBlock>

Le coefficient <MathInline>{"\\beta_1"}</MathInline> mesure la variation moyenne de <MathInline>{"y"}</MathInline> associée à une hausse d'une unité de <MathInline>{"x"}</MathInline>, toutes choses égales par ailleurs dans le cas multiple.
</MdxCard>

## Estimation par moindres carrés ordinaires

<MdxCard>
L'estimateur OLS minimise :

<MathBlock>{"\\min_{\\beta} \\sum_{i=1}^n (y_i - x_i^\\top \\beta)^2"}</MathBlock>

En notation matricielle, si <MathInline>{"X^\\top X"}</MathInline> est inversible :

<MathBlock>{"\\hat{\\beta} = (X^\\top X)^{-1} X^\\top y"}</MathBlock>

Cette formule montre immédiatement pourquoi la colinéarité pose problème : si les colonnes de <MathInline>{"X"}</MathInline> sont linéairement dépendantes, l'inverse n'existe pas.
</MdxCard>

## Propriétés théoriques

<MdxCard>
Sous les hypothèses usuelles, l'estimateur OLS est :

- **sans biais** : <MathInline>{"\\mathbb{E}[\\hat{\\beta} \\mid X] = \\beta"}</MathInline>
- **consistant** si la taille d'échantillon augmente dans un cadre régulier
- **BLUE** (Best Linear Unbiased Estimator) via le théorème de Gauss-Markov

Sa variance conditionnelle vaut :

<MathBlock>{"\\mathrm{Var}(\\hat{\\beta} \\mid X) = \\sigma^2 (X^\\top X)^{-1}"}</MathBlock>

Sous normalité des erreurs, OLS coïncide aussi avec l'estimateur du maximum de vraisemblance.
</MdxCard>

<MdxCard>
Les p-values ne suffisent pas à juger un modèle. Il faut aussi regarder :

- la taille et le signe des coefficients
- la stabilité hors échantillon
- la cohérence métier ou économique
- la sensibilité aux variables ajoutées ou retirées
</MdxCard>

## Limites

<MdxCard>
- La causalité ne découle pas d'une corrélation linéaire significative.
- Les outliers peuvent fortement déplacer les coefficients.
- La relation réelle peut être non linéaire ou dépendre d'interactions omises.
- En présence de fuite d'information ou de mauvaise temporalité, l'inférence devient trompeuse.
- La régression linéaire extrapole selon une droite ou un hyperplan, ce qui peut être économiquement absurde hors zone observée.
</MdxCard>

## Régression linéaire comme baseline ML

<MdxCard>
En machine learning tabulaire, la régression linéaire reste une **baseline essentielle** :

- rapide à entraîner
- très interprétable
- bon point de comparaison avant des modèles non linéaires
- utile pour détecter si la complexité supplémentaire (arbres, boosting, réseaux) apporte réellement quelque chose

Elle devient particulièrement pertinente quand les données sont peu nombreuses, le bruit modéré et l'interprétation des coefficients importante.
</MdxCard>
