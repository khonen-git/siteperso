# Random Forest

<MdxCard title="Définition">
La **forêt aléatoire** (Random Forest) est un ensemble d'**arbres de décision** entraînés sur des échantillons bootstrap des données, avec une **sous-sélection aléatoire de variables** à chaque division. Les prédictions finales agrègent les arbres par **vote majoritaire** (classification) ou **moyenne** (régression).
</MdxCard>

## Intuition

<MdxCard>
Un arbre seul capture des interactions non linéaires mais **sur-apprend** facilement (forte variance). La forêt aléatoire applique le **bagging** : entraîner de nombreux arbres sur des sous-échantillons différents puis les moyenner réduit la variance sans augmenter autant le biais.

La randomisation des features à chaque split **décorrèle** les arbres entre eux — condition nécessaire pour que l'agrégation améliore réellement la généralisation par rapport à un seul arbre profond.
</MdxCard>

## Fonctionnement

<MdxCard>
  <ol className="list-decimal pl-6 space-y-3">
    <li><strong>Bootstrap</strong> : pour chaque arbre <MathInline>b</MathInline>, tirer <MathInline>n</MathInline> observations avec remise parmi <MathInline>n</MathInline> (environ 63 % d'observations uniques par arbre).</li>
    <li><strong>Arbre non élagué</strong> : faire croître l'arbre jusqu'à une profondeur ou des critères d'arrêt fixés (<code>max_depth</code>, <code>min_samples_leaf</code>, etc.).</li>
    <li><strong>Random subspace</strong> : à chaque nœud, ne considérer qu'un sous-ensemble de <code>max_features</code> variables pour choisir le meilleur split.</li>
    <li><strong>Agrégation</strong> : classification → classe majoritaire ; régression → moyenne des prédictions des <MathInline>B</MathInline> arbres.</li>
  </ol>
</MdxCard>

<MdxCard>
**Score out-of-bag (OOB)** : les observations non incluses dans le bootstrap d'un arbre servent d'échantillon de validation implicite. Avec <code>oob_score=True</code>, on obtient une estimation de performance sans jeu de validation dédié — utile en première passe, mais pas un substitut à une [validation croisée](/knowledge/machine-learning/general-concepts/cross-validation) rigoureuse.
</MdxCard>

## Forces et limites

<div className="grid gap-4 md:grid-cols-2">

<MdxCard>

### Forces

- Robuste aux outliers et aux échelles de variables (pas de normalisation obligatoire)
- Capture interactions et non-linéarités sans feature engineering explicite
- Importance des variables intégrée ; score OOB gratuit
- Parallélisable ; relativement peu d'hyperparamètres critiques

</MdxCard>

<MdxCard>

### Limites

- Moins interprétable qu'un arbre unique
- Coût mémoire et latence croissants avec <code>n_estimators</code>
- Régression : mauvaise extrapolation hors du domaine d'entraînement
- Peut être dominé par le [gradient boosting](/knowledge/machine-learning/classical-models/xgboost) sur données tabulaires structurées
- Sensible au **concept drift** et au **non-iid** sans adaptation

</MdxCard>

</div>

## Applications

<div className="grid gap-4 md:grid-cols-2">

<MdxCard>

### Classification tabulaire

Scoring crédit, détection de fraude, classification multi-classe sur features hétérogènes — cas d'usage historique des forêts aléatoires.

</MdxCard>

<MdxCard>

### Régression

Prédiction de valeurs continues (prix, volumes) ; attention à l'extrapolation hors du support d'entraînement.

</MdxCard>

<MdxCard>

### Finance et marchés

Modèle de base pour prédire un label (direction, régime) à partir de features techniques ; souvent surpassé par le boosting mais utile comme benchmark robuste et pour l'importance des variables.

</MdxCard>

<MdxCard>

### Feature screening

Identifier rapidement les variables pertinentes avant un modèle plus complexe ou un pipeline de [feature engineering](/knowledge/machine-learning/general-concepts).

</MdxCard>

</div>

from sklearn.ensemble import RandomForestClassifier
from sklearn.model_selection import cross_val_score
from sklearn.inspection import permutation_importance

X_train, y_train = ...  # features tabulaires, labels

rf = RandomForestClassifier(
    n_estimators=200,
    max_depth=12,
    min_samples_leaf=5,
    max_features="sqrt",
    class_weight="balanced",
    oob_score=True,
    n_jobs=-1,
    random_state=42,
)
rf.fit(X_train, y_train)

print(f"OOB score : {rf.oob_score_:.3f}")
cv_scores = cross_val_score(rf, X_train, y_train, cv=5, scoring="f1_macro")
print(f"CV F1 macro : {cv_scores.mean():.3f} (+/- {cv_scores.std():.3f})")

# Importance par impureté
for name, imp in sorted(
    zip(feature_names, rf.feature_importances_),
