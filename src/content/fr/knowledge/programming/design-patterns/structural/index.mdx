# Patterns structurels

<MdxCard>
Les patterns **structurels** expliquent comment assembler objets et classes en structures plus larges, tout en gardant le système flexible et efficace.
</MdxCard>

## Liste

| Pattern | Idée clé |
|---------|----------|
| [Adapter](/knowledge/programming/design-patterns/structural/adapter) | Interface incompatible → interface attendue |
| [Bridge](/knowledge/programming/design-patterns/structural/bridge) | Décorréler abstraction et implémentation |
| [Composite](/knowledge/programming/design-patterns/structural/composite) | Arbre partiel-total |
| [Decorator](/knowledge/programming/design-patterns/structural/decorator) | Ajouter des responsabilités dynamiquement |
| [Facade](/knowledge/programming/design-patterns/structural/facade) | API simple sur un sous-système |
| [Flyweight](/knowledge/programming/design-patterns/structural/flyweight) | Partager l'état intrinsèque |
| [Proxy](/knowledge/programming/design-patterns/structural/proxy) | Substitut contrôlant l'accès |
