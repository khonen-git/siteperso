Coroutines et tâches

Les coroutines et les tâches sont des éléments fondamentaux d'asyncio, permettant de réaliser des opérations asynchrones.

Coroutines

Les coroutines sont des fonctions définies avec async def et sont le moyen de créer une unité de travail asynchrone. Elles peuvent être mises en pause et reprises à l'aide du mot-clé await. 


import asyncio

async def coroutine_example():
    print("Début de la coroutine")
    await asyncio.sleep(1)
    print("Fin de la coroutine")

Dans cet exemple, la coroutine coroutine_example imprime un message, s'endort pendant 1 seconde (simulant une opération asynchrone), puis imprime un autre message.

Tâches

Les tâches sont utilisées pour planifier l'exécution des coroutines de manière concurrente. Elles enveloppent les coroutines et les gèrent, permettant ainsi à plusieurs coroutines de s'exécuter simultanément. 


async def main():
    # Création d'une tâche pour exécuter la coroutine
    task = asyncio.create_task(coroutine_example())
    
    print("Autres opérations pendant que la coroutine est en pause")
    await task  # Attente de la fin de la tâche

asyncio.run(main())

Dans cet exemple, asyncio.create_task() est utilisé pour créer une tâche à partir de la coroutine coroutine_example. La fonction main continue de s'exécuter après la création de la tâche, illustrant la concurrence. L'attente de la tâche avec await task garantit que main attend que la coroutine enveloppée dans la tâche soit terminée avant de terminer.

Enchaînement de Coroutines

On peut également enchaîner des coroutines, c'est-à-dire en exécuter une après l'autre. 


async def first_coroutine():
    print("Première coroutine")
    await asyncio.sleep(1)
    return "Résultat 1"

async def second_coroutine():
    print("Seconde coroutine")
    await asyncio.sleep(2)
    return "Résultat 2"

async def main():
    result1 = await first_coroutine()
    result2 = await second_coroutine()
    print(result1, result2)

asyncio.run(main())

Ici, main attend d'abord que first_coroutine se termine, puis exécute second_coroutine. Cette approche garantit un ordre d'exécution spécifique.
